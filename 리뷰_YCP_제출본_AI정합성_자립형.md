# YCP 제조AI 특화 스마트공장 사업신청서 (제출본) AI 정합성 검토 — 자립형 보고서

> **본 보고서의 성격** — 본 보고서는 YCP 가 제출한 「제조AI 특화 스마트공장 사업신청서」(이하 "제출본") 41 페이지 본문에 대한 사후 정합성 검토 결과이다. 본 보고서는 **단일 자립형 문서**로, 별도 참고 자료를 열람하지 않고도 (i) 정합성 진단 결과 (ii) 보충해야 할 누락 사항 (iii) 수정해야 할 부분 (iv) 본문에 부재한 인력 M/M 산정안 (v) 우선순위 권고 (vi) 차기 신청서 적용 체크리스트 모두를 이해할 수 있도록 작성되었다.
>
> **작성 목적** — 제출본의 사후 검토 결과를 사업신청서 작성 담당자에게 전달하여, 차기 사업신청 (동일 사업의 재공모 또는 후속 사업) 시 본 보고서에서 도출된 권고 16 항목 + 점검 체크리스트 25 항목을 직접 적용 가능하도록 한다.
>
> **사용 가이드** — §1 검토 개요 → §6 우선순위 권고 (Critical 4 / High 6 / Medium 6) → §7 차기 신청서 체크리스트 순으로 빠르게 훑은 후, 관심 영역의 §2~§5 본문을 정독하는 것을 권장한다. §5 인력 M/M 산정안은 본 보고서의 핵심 산출물로, 제출본에 부재한 M/M 표를 신규로 제시한 부분이다.

## §2. AI 섹션 정합성 진단

### 2.1 AI 방법론 표준 5 항목 평가

본 절은 일반 제조 AI 사업신청서가 갖춰야 할 **AI 방법론 표준 5 항목**을 기준으로 제출본의 명시 정도를 평가한다. 5 항목과 그 표준 정의는 다음과 같다.

#### 2.1.1 표준 5 항목의 정의

**(1) 데이터 유형 분류** — 사업 대상 공정에서 수집·활용 가능한 데이터를 (i) PLC·SCADA·Historian 등에서 수집하는 시계열 신호 (ii) MES·QMS·ERP 등 정형 데이터베이스 (iii) 비전 카메라·CCTV 등 이미지 스트림 (iv) 공정설계서·SOP·작업표준서·Mill Sheet 등 비정형 문서의 4 분류로 명시. 각 분류별 보유 규모·수집 주기·구축 위치·디지털화 필요 여부를 표 형태로 제시.

**(2) 피쳐 엔지니어링 전략 — 3 축 + 피쳐 스토어** — AI 모델의 입력 피쳐를 다음 3 축으로 체계화:
- *축 1: 도메인 지식 기반 피쳐* — 공정의 물리적 특성을 반영한 패스 이력 누적값, 슬라이딩 윈도우 통계 (평균·표준편차·최소·최대), 공정 구간 차분, 재질·레시피 메타 결합 등. 현장 숙련자의 암묵지를 정량화한 것.
- *축 2: 자동 피쳐 생성* — `tsfresh`·`featuretools` 등 시계열 피쳐 자동 추출 라이브러리 + 도메인 전문가가 인지하지 못한 잠재 피쳐 후보 풀 (수백~수천 개) 확보.
- *축 3: 다단계 피쳐 선정* — (a) 상관관계 분석으로 다중공선성 제거 → (b) 상호정보량 (Mutual Information) 으로 비선형 관계 평가 → (c) SHAP (Shapley Additive Explanations) 으로 모델 기여도 분석 → 통계적·모델 기반 양 측면에서 의미 있는 피쳐만 채택.
- *피쳐 스토어 등재* — 학습/추론 시점의 피쳐 정의를 일관되게 관리하여 학습-추론 간 불일치 (training-serving skew) 방지 + 신규 시나리오 도입 시 즉시 재활용 가능한 형태로 등재.

**(3) 모델 선정 거버넌스 — 5 객관 기준 + 3 단계 절차** — 단일 알고리즘 의존 대신 문제 유형별 모델 후보군을 사전 정의하고 다음 5 객관 기준에 따라 채택:
- *데이터 규모* — 라벨 보유량·세션 길이·표본 다양성
- *해석가능성* — 심사·현장 수용성·규제 대응 관점에서 SHAP·Attention 등 설명 도구 적용 가능성
- *추론 지연* — 실시간 제어 요구 시 100ms 이하 보장 가능 여부
- *재학습 주기* — 데이터 드리프트 빈도·라벨 수집 주기를 고려한 재학습 비용
- *엣지 배포 가능성* — GPU·NPU 가용 자원·OS 제약 부합 여부

선정 절차는 (a) 베이스라인 모델 (통상 XGBoost 또는 단순 통계 모델) → (b) 후보 모델 다중 학습·교차검증 → (c) 채택 모델 결정의 3 단계로 진행.

**문제 유형별 표준 모델 후보군**:
- 회귀 (품질 수치 예측): XGBoost, LightGBM
- 시계열 예측 (공정 추이): LSTM, Transformer, TCN
- 이상탐지 (설비 건전성): Isolation Forest, AutoEncoder, OneClassSVM
- 분류 (비전 결함, 문서): EfficientNet, ViT (비전), Transformer 계열 (문서)
- 추천 (유사 사례, 레시피): 유사도 기반 Retrieval, LLM 결합 RAG

**(4) 앙상블 정당화 — 3 패턴 중 선택** — 단일 모델로 충분한 성능을 확보하기 어려운 경우 다음 3 앙상블 패턴 중 선택·조합:
- *Stacking* — 예측값을 메타 모델 입력으로 재학습
- *Weighted Average* — 검증 성능 기반 가중치 결합
- *Model Router* — 입력 특성에 따라 적합한 전문 모델로 분기

**(5) 엔드투엔드 파이프라인 — 8 단계** — 다음 8 단계를 단일 흐름으로 통합:
1. 데이터 수집 (시계열 + 정형 + 이미지 + 비정형 문서를 데이터 레이크 구조에 적재)
2. 정제·라벨링 (결측·이상치·중복·단위 불일치 처리 + 검사 결과·정비 이력 라벨 결합)
3. 피쳐 엔지니어링 (다단계 선정 후 피쳐 스토어 등재)
4. 학습·평가 (베이스라인→후보→채택 3 단계 평가)
5. 모델 레지스트리 (버전·메타데이터·성능 지표 등록)
6. 배포 (엣지 노드 또는 서버, 추론 지연 요건별)
7. 운영·실시간 추론 (HMI 또는 MES·SCADA 화면 통합)
8. 운영 피드백·재학습 루프 (HITL 검증 결과·실측 라벨 환류 → 드리프트 탐지 → 자동 재학습)

#### 2.1.3 5 항목 종합 평가

★ 2 항목 (데이터 유형 분류·엔드투엔드 파이프라인) / ◐ 1 항목 (앙상블) / ○ 2 항목 (피쳐 엔지니어링·모델 선정 거버넌스) / △ 0 항목 = 평균 ◐ 수준.

**강점 2 항목**:
- 데이터 보유 매트릭스의 디테일 — 16 공정 × 8 속성 표는 표준 이상의 충실도. 차기 사업에서도 동 표 양식 재사용 가능.
- 엔드투엔드 파이프라인 구성 — 5 단계 + MLOps 풀 도입 + 재학습 트리거·A/B·롤백·월별 리포트 모두 정량 기준.

**약점 2 항목 (즉시 보강 필요)**:
- 피쳐 엔지니어링 전략 부재 — 단위 통일만 명시, 도메인 피쳐·자동 생성·다단계 선정·피쳐 스토어 모두 부재. AI 모델의 입력 품질이 모델 성능을 직접 좌우하는데 본 영역이 누락되면 심사자가 "AI 모델이 단순 raw 데이터 입력의 블랙박스" 로 판단 가능.
- 모델 선정 거버넌스 부재 — 모델 후보군 비교, 5 객관 기준 평가, 3 단계 선정 절차 모두 미서술. 특히 LLM 의 외부 API (OpenAI 명시) vs 사내 sLM 분기 부재가 영업비밀·OEM 사양 보호 관점에서 보안 리스크.

### 2.2 AI 엔진 패턴 매핑

본 절은 제조 AI 사업에서 활용되는 **AI 엔진 패턴 7 종**의 표준 분류를 본문 인라인으로 제시하고, 제출본의 AI 시스템이 어느 패턴에 해당하는지 매핑한다.

#### 2.2.1 AI 엔진 패턴 7 종의 정의

**패턴 A — 유사 사례 검색·추천 엔진**
- 적용: 신규 주문·작업 투입 시 과거 성공 이력 (사양·설정·결과) 을 지식베이스로 축적하고 유사 사례 기반 초기 설계 초안을 자동 생성하여 숙련자 의사결정 보조.
- 엔진 구조: (i) 유사 케이스 검색 모듈 — 재질·치수·형상 피쳐를 결합한 벡터 임베딩 + 조건 필터 (재질군·치수 허용 범위) (ii) 추천 모듈 — Top-N 과거 사례를 통계 집계 또는 LLM 요약으로 정리해 추천 레시피 N 개 제시 (iii) 규칙 검증 모듈 — 허용 압하율·설비 용량·열처리 온도 범위·공차 한계 등 물리·설비 제약 체크 (iv) 출력 정보 — 추천 공정 경로·조건 범위·참조 유사 오더·예상 수율·품질.
- 선행조건: 과거 이력의 구조화 (재질·치수·공정 파라미터·품질 결과 일관 스키마), 자유 텍스트 주석은 OCR·정제 후 편입, 숙련자 검수 루프 (HITL) 와 필수 결합.
- 대표 사용처: 단조·절단·절곡 설정 추천, 인발·필거밀 공정설계, 공구·금형 관리.

**패턴 B — 시계열 품질·이탈 예측 엔진**
- 적용: 공정 시계열 신호를 실시간 추론해 품질 이탈을 사전에 예측·경보하고, 필요 시 피드포워드 조치 힌트를 작업자에게 제공.
- 엔진 구조: (i) PLC·Historian → 스트림 버퍼 (초·10Hz·100Hz 등 혼합) (ii) 슬라이딩 윈도우 통계, 스탠드·설비별 기여도 (iii) 1D-CNN / LSTM / Transformer 선택 (iv) 이탈 판정 모듈 — 목표 대비 σ 임계 + 추세 기반 조기 경보 트리거 (v) HMI 경보 + 조작 변수 제안값.
- 선행조건: PLC 태그 표준화·시간 동기 (NTP), 목표 품질 실측 라벨 확보, 추론 지연 요구 확정 (실시간 제어 시 100ms 이하).
- 대표 사용처: 연속주조 품질, 열연·냉연 두께, 사출 불량, 공구 마모.

**패턴 C — 비전 검사 엔진**
- 적용: 이미지·영상·포인트클라우드 기반 비전 AI 로 결함 검출, 치수 측정, 행동 인식 등을 자동화하여 육안 검사 한계 극복.
- 엔진 구조: (i) 촬영·조명 설계 — 라인스캔 / 에어리어 / 다각도 / 3D 구조광 선택 (ii) 라벨링·사전학습 — 소량 라벨 + Self-supervised Pretraining 또는 Synthetic Data 보강 (iii) 모델 — 분류 (EfficientNet·ViT), 탐지 (YOLO·DETR), 세그멘테이션 (U-Net·Mask2Former) (iv) 후처리 — 결함 등급 매핑, CAD·설계 허용치 대비 편차 산출 (v) 엣지 배포.
- 선행조건: 조명·이송 속도 고정 (조명 변동은 모델 재학습 원인 1 위), CCTV·작업자 촬영 시 노사 합의·마스킹.
- 대표 사용처: 표면결함 비전, UT/ECT 자동판정 (신호 이미지화), 용접 비드, 3D 치수 검사.

**패턴 D — 예지보전 엔진**
- 적용: 설비 건전성 (Health) 을 지속 감시하고 이상 징후를 조기 탐지해 잔여수명 (RUL) 을 추정함으로써 과잉 정비와 돌발 고장의 양극단을 동시에 회피.
- 엔진 구조: (i) 진동 가속도·속도, 모터 전류·전압, 윤활유 온도·압력, AE (Acoustic Emission) 수집 (ii) FFT / Envelope / Cepstrum 특징 추출 (iii) Autoencoder · Isolation Forest · OneClassSVM — 정상 상태 학습 기반 이상탐지 (iv) RUL 추정 — Survival Analysis, LSTM Regression (v) CMMS 연동 — 임계 초과 시 워크오더 자동 생성.
- 선행조건: 진동·전류 고주파 수집 인프라 (≥ 1kHz 권장), 기계별 고장 모드 도메인 지식, 정비 이력 정제.
- 대표 사용처: 압연기 예지보전, CNC 공구, 컴프레서·보일러 누기 감시.

**패턴 E — 공정 최적화·제어 엔진**
- 적용: 조작 변수 공간에서 목적 함수 (수율·에너지·사이클 타임) 를 최적화하는 추천·제어 엔진. 예측을 넘어 의사결정을 출력.
- 엔진 구조: (i) 환경 모델링 — 물리 기반 + 데이터 기반 하이브리드 (Process Model · Digital Twin) (ii) 베이지안 최적화 (BO), 강화학습 (RL), 물리 제약 통합 수학 최적화 (MILP·NLP) (iii) Safe RL · 제약 BO — 허용 범위 초과 제안 차단 (iv) 오픈루프 (작업자 승인) 또는 클로즈드루프 (DCS 연동).
- 선행조건: RL 은 시뮬레이터 또는 오프라인 안전 평가 체계 선행, 제어 통합 시 책임·법적 이슈 + 안전 PLC 연동.
- 대표 사용처: 전기로 (EAF) 전력·전극 최적화, 소둔 적재·온도 프로파일, 도금 조건, 가류 시간·온도.

**패턴 F — LLM·RAG 지식검색 엔진** (제출본 핵심 패턴 1)
- 적용: 비정형 기업 지식 (문서·도면·이력) 을 LLM 과 검색 엔진으로 연결해 현장 실무자의 질문에 근거 제시 답변 또는 문서 초안을 생성.
- 엔진 구조 — **5 계층**:
  - *계층 1: 문서 수집·정제* — SOP·매뉴얼·CMMS 이력·도면·MSDS·Mill Sheet 를 PDF·HWP·DWG·이미지 OCR 파서로 추출
  - *계층 2: 청킹·임베딩* — 문서 특성별 전략 (계층 청킹·섹션 기반·토픽 기반), 멀티뷰 임베딩
  - *계층 3: 검색기* — 하이브리드 (Dense + BM25 + 메타 필터), Re-ranker 로 상위 정제
  - *계층 4: LLM 응답* — 근거 문서·페이지·문단 인용 (Citation 강제), 환각 방지 정책, 확인 필요 시 휴먼 에스컬레이션
  - *계층 5: 권한·보안 라우팅* — 문서 접근권한 동기화, 민감 정보 마스킹, 외부 LLM 여부에 따른 온프레미스·하이브리드 라우팅
- 선행조건: 문서 디지털화·표준화가 가장 큰 선행 작업, 민감도 평가 후 외부 LLM 사용 가능 여부 사전 결정, 인사·계정 시스템 권한과의 연동.
- LLM 모델 선택지:
  - *외부 API*: OpenAI GPT-4·GPT-4o, Anthropic Claude, Google Gemini — 한국어 성능 우수하나 영업비밀 전송 리스크
  - *사내 호스팅 (한국어 sLM)*: LG EXAONE 시리즈, 네이버 HyperCLOVA X, 삼성 Gauss, KT Mi:dm — 한국어 성능 + 보안 양립
  - *자체 호스팅 (오픈소스)*: Llama·Qwen·Solar 등 자체 GPU 서버 호스팅 — 가장 강한 보안, 운영 부담 큼
- 대표 사용처: SOP RAG, 장애 대응 RAG, 불량 보고서 자동 작성, 도면 지능 검색, Mill Sheet OCR.

**패턴 G — 도면·CAD 형상 임베딩 엔진**
- 적용: CAD 도면 (DWG/DXF/STEP/IFC) 의 기하·형상 정보를 형상 벡터 임베딩으로 변환하여, 텍스트·메타 검색이 닿지 않는 형상 유사 검색·치수 비교·제품군 분류 가능. 패턴 F 텍스트 모드와 병기 시 "이름이 비슷한 부품" 과 "모양이 비슷한 부품" 동시 검색.
- 엔진 구조: (i) 도면 수집·파싱 — DWG/DXF (AutoCAD 라이브러리·ODA File Converter), STEP/IGES/IFC (OpenCASCADE) (ii) 형상 표현 변환 — 3D 메쉬 ↔ 포인트클라우드 ↔ B-Rep ↔ 복셀 (iii) PointNet++/DGCNN (포인트클라우드), MeshCNN (메쉬), GraphSAGE (B-Rep) 또는 멀티뷰 렌더링 → CNN (iv) 형상 검색 — 형상 벡터 코사인 유사도 + 메타 필터 (v) CAD 비교 모드 — 신규 ↔ 표준 도면의 형상·치수 차분 (ICP 정합 후 잔차 분석).
- 선행조건: CAD 라이브러리 라이선스 확보, 도면의 IP·영업비밀 민감도가 패턴 F 보다 높음 — 외부 임베딩 API 사용은 정밀 도면에 부적합.
- 대표 사용처: CAD 도면 지능 검색, 3D 치수 검사, 단조·절단·절곡 설정 추천.

## §3. 보충해야 할 부분

본 절은 제출본이 일반 사업신청서 표준 구성에서 누락한 항목을 6 영역으로 정리한다. §4 (수정) 와 달리 **이미 명시된 내용을 고치는 것이 아니라, 부재한 섹션·정보를 새로 추가하는 작업**이다.

### 3.1 데이터 품질 검증 메커니즘 (5 항목)

제출본은 약 7,000 건 엑셀 데이터 + 1,200 건 OCR 처리 + 800 건 Mill Sheet 추가 + 5 년 누적 4,500 건 등 약 13,500 건의 데이터를 활용한다고 명시한다 (p.27, p.40). 그러나 **데이터의 품질 검증·정제·라벨링·이상치 탐지·표본 다양성 평가** 의 5 항목 메커니즘이 부재하다.

**일반 표준 — 데이터 품질 검증 5 항목**:

1. **이상치 탐지 룰셋** — 학습 데이터의 단위 불일치 (예: 외경값에 mm 와 inch 혼재), 결측 (필수 항목 누락), 중복 (동일 Heat No. 가 다른 화학 성분으로 두 번 등록), 범위 초과 (열처리 온도가 1500°C 표기 등 물리적 불가능 값) 를 자동 탐지하는 표준 룰셋. 일반적으로 IQR (사분위 범위) · Z-score · Isolation Forest 등의 통계적 방법 + 도메인 룰 (재질별 허용 범위) 결합.

2. **라벨 정확도 검증** — 과거 설계 결과의 "성공 / 부분 수정 / 폐기" 라벨이 어떻게 부여되는지 명시 + 검수자 간 일치율 측정 절차 (Cohen's Kappa coefficient 또는 Krippendorff's alpha 가 표준). 두 명 이상의 검수자가 동일 데이터를 독립 라벨링한 후 일치율 0.8 이상 (Substantial agreement) 을 검증 통과 기준으로 사용.

3. **데이터 표본 다양성** — 13,500 건이 강종별 분포 (SUS304·SUS316L·기타 강종), OEM·고객사별 분포 (자동차·플랜트·조선), 모관 사이즈 분포 (외경 5~50mm 등) 가 어떤지. **표본 편중 시 모델이 소수 강종·소수 OEM 에 대해 일반화 실패 위험**. 대응 방법으로 (a) 계층 샘플링 (Stratified Sampling) 으로 학습/검증 분할 (b) 클래스 가중치 조정 (Class Weight) (c) 데이터 증강 (Data Augmentation) 적용.

4. **테스트셋 분리 정책** — 학습/검증/테스트 데이터 분리 비율 (일반적 70:15:15 또는 80:10:10), 시간 분할 (Time-based split — 시계열 데이터에 필수) 또는 무작위 분할 (Random split) 선택 사유 명시. 공정설계 데이터는 시간에 따른 공정 표준 변경·OEM 요구 변경이 있으므로 **시간 분할이 일반적으로 권장** — 학습 데이터는 5 년 전 ~ 1 년 전, 검증은 1 년 전 ~ 6 개월 전, 테스트는 최근 6 개월 데이터.

5. **OCR 정확도 검증 절차** — 제출본 p.30 의 "공급사별 인식 정확도" 모니터링 명시는 기본 기능. 추가로 (a) 인식 결과의 실제 검증 — 담당자 수기 대조의 표본 추출 방법 (예: 매주 무작위 50 건 추출 후 정확도 ≥ 95% 검증) (b) 빈도 — 운영 초기 6 개월 매주, 안정화 후 월 1 회 (c) 실패 케이스 RCA — 인식 실패 5% 의 원인 분류 (스캔 품질·필기 처리·표 구조 비정형 등) 와 대응 정책.

**보충 권고 — 본문 §1.1 에 다음 표 추가**:

| 데이터 품질 검증 항목 | 적용 방법 | 검증 빈도 | 통과 기준 |
|---|---|---|---|
| 이상치 탐지 | IQR + 도메인 룰 (재질별 압하율·열처리 온도 범위) | 데이터 적재 시 자동 | 이상치 비율 < 1% |
| 라벨 일치율 | Cohen's Kappa (검수자 2 명) | 분기 1 회 표본 100 건 | Kappa ≥ 0.8 |
| 표본 다양성 | 강종·OEM·사이즈 분포 분석 | 학습 트리거 시 | 최소 클래스 / 최대 클래스 ≥ 0.1 |

**항목 1: 임베딩 모델 선택 사유**

제출본은 "Vector DB 임베딩" 만 명시하나, 임베딩 모델의 선택은 RAG 성능을 직접 좌우한다. 한국어 + 도메인 (특수강관) 의 임베딩 모델 후보:

| 임베딩 모델 | 한국어 성능 | 도메인 파인튜닝 | 운영 환경 | 라이선스 |
|---|---|---|---|---|
| OpenAI text-embedding-3 | 우수 | API 호출 | 외부 API | 유료 (사용량 기반) |
| KoSimCSE | 양호 | 가능 | 사내 호스팅 | 오픈소스 (MIT) |
| KR-SBERT | 양호 | 가능 | 사내 호스팅 | 오픈소스 |

| E5-large | 우수 | 가능 | 사내 호스팅 | 오픈소스 |

선택 기준: (a) 영업비밀·OEM 사양 보호 → 사내 호스팅 강제 → KR-SBERT·KoE5·E5-large 후보 (b) 일반 SOP·공정 지식 → OpenAI 사용 가능 (c) 도메인 파인튜닝 — 특수강관 도메인 어휘 (모관·필거·인발·압하율 등) 학습 효과 큼. **권고: KR-SBERT 또는 KoE5 를 1 차 채택 + 1,200 건 OCR 처리 데이터로 도메인 파인튜닝.**

**항목 2: 하이퍼파라미터 탐색 전략**

모델별 하이퍼파라미터 탐색 방법 명시:

| 모델 유형 | 권장 탐색 방법 | 탐색 공간 |
|---|---|---|
| XGBoost·LightGBM (regression) | Bayesian Optimization | learning_rate, max_depth, n_estimators, min_child_weight |
| Transformer (LLM 파인튜닝) | Random Search | learning_rate, batch_size, warmup_steps, weight_decay |
| 임베딩 모델 (대조학습) | Grid Search | margin, temperature, batch_size |

**항목 3: 교차검증 절차**

특수강관 데이터의 강종·OEM별 편중을 고려하여 **계층 K-Fold (Stratified K-Fold, k=5)** 권장:
- 각 폴드에 강종·OEM 분포가 전체 데이터와 동일하도록 분할
- 5 폴드의 평균 성능 + 표준편차로 모델 안정성 평가
- 시계열 효과가 있다면 Time Series Split (TSS) 추가 적용

**항목 4: RAG 평가 KPI**

제출본은 "AI 추천 정확도 ≥ 85%" 명시하나 정확도의 정의 미명시. RAG 시스템의 일반 평가 KPI:

| KPI | 정의 | 목표값 (일반) |
|---|---|---|
| Recall@K (Retrieval) | 검색 결과 상위 K 개에 정답 포함 비율 | Recall@5 ≥ 0.85 |
| MRR (Mean Reciprocal Rank) | 정답의 평균 역순위 | MRR ≥ 0.7 |
| NDCG@K | 순위를 고려한 검색 품질 | NDCG@5 ≥ 0.8 |
| Faithfulness (Generation) | LLM 답변이 검색 근거에 충실한 정도 | ≥ 0.9 (LLM-as-judge) |
| Answer Relevance | LLM 답변이 질문에 적절한 정도 | ≥ 0.85 |

### 3.3 암묵지 추출 프로세스 구체화 (4 항목)

제출본의 핵심 명분은 "1~2 명 숙련공의 20 년 경험 (암묵지) 을 회사 영구 디지털 자산으로 전환" 이다 (p.27). 그러나 **암묵지를 어떻게 추출·구조화·임베딩하는지** 의 프로세스가 추상적이다 — "OCR 로 1,200 건 자동 추출" + "NLP 로 정형화 후 Vector DB 임베딩" 만 명시 (p.41).

**암묵지 추출의 본질적 문제** — OCR 로는 베테랑이 **이미 종이·엑셀에 기록한** 명시적 정보만 추출 가능. 베테랑의 머릿속에 있는 의사결정 사유 (왜 그 패스 횟수를 선택했는지, 왜 그 열처리 온도를 선택했는지, 어떤 재질·OEM 조합에 어떤 특이사항을 적용했는지) 는 OCR·NLP 로 추출 불가능. 별도의 **베테랑 인터뷰 프로세스**가 필요하다.

**보충 권고 — 다음 4 항목 본문에 추가**:

**항목 1: 베테랑 인터뷰 양식 (반구조화 인터뷰)**

| 인터뷰 단계 | 내용 | 시간 | 산출물 |
|---|---|---|---|
| 1. 사전 조사 | 인터뷰 대상 베테랑의 과거 5 년 작성 공정설계서 30~50 건 사전 분석 | 2~3 일 (분석자) | 인터뷰 가이드 + 시나리오 카드 |
| 2. 1 회차 인터뷰 (~2 시간) | 일반 의사결정 패턴 — "신규 주문이 들어오면 어떤 순서로 검토하는가" | 2 시간 | 음성 녹음 + 스케치 메모 |
| 3. 2 회차 인터뷰 (~2 시간) | 특수 케이스 — "지난 1 년 가장 어려웠던 5 건 + 의사결정 사유" | 2 시간 | 케이스별 의사결정 트리 |
| 4. 3 회차 인터뷰 (~1.5 시간) | 실패·재작업 케이스 — "재작업이 발생한 케이스 + 학습 포인트" | 1.5 시간 | 실패 케이스 패턴 |
| 5. 검증 인터뷰 (~1 시간) | 추출된 의사결정 트리·휴리스틱을 베테랑이 검증·수정 | 1 시간 | 최종 휴리스틱 룰셋 |

베테랑 1 명당 총 6.5 시간 + 분석자 사전 조사 2~3 일. 베테랑 1~2 명 가정 시 총 13~30 시간 인터뷰 + 6 일 분석.

**항목 2: 휴리스틱 구조화 방식**

베테랑 의사결정 근거의 자료 구조 선택 기준:

| 구조 | 적합 케이스 | 예시 |
|---|---|---|
| 의사결정 트리 (Decision Tree) | 명시적 if-then 규칙이 다수 | "재질이 SUS304 이고 외경 ≥ 30mm 이면 인발 패스 2 회 + 열처리 1080°C" |
| 룰 엔진 (Rules Engine — Drools 등) | 다수 룰 + 우선순위 + 충돌 해결 | "OEM 별 특이 룰 우선 → 일반 룰 후순위" |
| 케이스 베이스 (Case-Based Reasoning) | 유사 사례 검색 + 수정 | "과거 OEM A 의 유사 주문 3 건 → 해당 케이스 파라미터 + 수정" |
| Vector + 메타 (RAG) | 비정형 + 유사도 검색 | "외경·내경·재질 임베딩 + Top-N 검색 + LLM 해설" |

**권고**: 의사결정 트리 (명시적 룰) + RAG (유사 사례) 의 **하이브리드 구조**. 단순 룰은 의사결정 트리에, 복잡한 패턴은 RAG 에 분리 저장.

**항목 3: 임베딩 청킹 전략**

13,500 건 데이터의 다양한 문서 타입에 대한 청킹 전략:

| 문서 타입 | 권장 청킹 전략 | 청크 크기 | 이유 |
|---|---|---|---|
| 공정설계서 (엑셀 7,000 건) | 행 단위 청킹 | 단일 행 (~50~100 토큰) | 각 행이 1 건 주문의 완결된 설계 정보 |
| Mill Sheet (1,200 건 + 800 건) | 표 단위 청킹 | 1 Mill Sheet 전체 (~200~400 토큰) | 화학 성분·기계 성질·Heat No. 가 한 단위로 의미 |
| SOP·작업표준서 | 섹션 기반 청킹 | 섹션당 ~512 토큰 | 절차의 단위 = 섹션 |
| 베테랑 인터뷰 녹취 | 토픽 기반 청킹 | 토픽당 ~256~512 토큰 | 의사결정 사유 단위 |
| 작업일지 | 일자 + 사건 단위 | ~100~300 토큰 | 일자별 사건이 검색 단위 |

**항목 4: 자산화 KPI 측정 방식**

제출본의 "지식 자산화율 0% → 100%" KPI 의 정의·측정 수식:

```
자산화율 = (RAG 색인된 의사결정 사유 건수) / (베테랑이 지난 5 년간 작성한 공정설계서 건수)
```

분자 = (i) OCR 처리된 1,200 건 + (ii) NLP 정형화 후 Vector DB 색인된 의사결정 사유 (인터뷰 추출 휴리스틱 포함) 건수의 합. 분모 = 5 년 누적 공정설계서 (제출본 §9 표상 약 4,500 건) 건수. **목표 100% 달성을 위해서는 4,500 건의 의사결정 사유가 모두 RAG 색인되어야 함** — 1,200 건은 OCR, 나머지 3,300 건은 NLP 패턴 매칭 또는 추가 인터뷰로 보충 필요.

### 3.5 위험 관리 (3×3 매트릭스)

제출본은 "지식 휘발 리스크" 만 정성적으로 언급한다 (p.16, p.21). 그러나 **위험 3×3 매트릭스 (발생 가능성 × 영향도) + 4 카테고리 + 대응 전략** 이 부재하다.

**일반 표준 — 위험 4 카테고리 + 3×3 매트릭스**:

**위험 4 카테고리**:
- (i) **기술적 위험** — 모델 성능 미달, 데이터 부족·품질 저하, 인프라 호환성 (예: GPU·MES API), 알고리즘 한계
- (ii) **사업적 위험** — 인력 이탈 (특히 베테랑·핵심 DS), 일정 지연, 예산 초과, 사업 범위 변경
- (iii) **운영적 위험** — 현장 수용성 (작업자 거부), HITL 피로도, 시스템 안정성, 신뢰도 부족
- (iv) **외부 위험** — 규제 변경 (CBAM·중대재해법), OEM 요구 변경, 클라우드 공급자 정책 변경, 보안 사고

**위험 3×3 매트릭스**:

| 발생 가능성 \ 영향도 | 낮음 (1) | 중간 (2) | 높음 (3) |
|---|---|---|---|
| 높음 (3) | 모니터링 (3) | 대응 계획 수립 (6) | 즉시 완화 조치 (9) |
| 중간 (2) | 모니터링 (2) | 대응 계획 수립 (4) | 대응 계획 + 완화 (6) |
| 낮음 (1) | 수용 (1) | 모니터링 (2) | 대응 계획 수립 (3) |

각 셀의 숫자 = 위험 점수. 점수 ≥ 6 이면 즉시 대응, 3~5 는 대응 계획, 1~2 는 모니터링.

**제조 AI 사업 일반 주요 위험 (예시)**:

| 위험 | 카테고리 | 발생 가능성 | 영향도 | 점수 | 대응 전략 |
|---|---|:---:|:---:|:---:|---|
| 베테랑 1~2 명 퇴직·이직 | 사업적 | 중간 | 높음 | 6 | 베테랑 보너스 계약 + 인터뷰 6 회 우선 진행 + 퇴직 6 개월 전 통보 의무 계약 |
| AI 모델 정확도 85% 미달 | 기술적 | 중간 | 높음 | 6 | 베이스라인 70% 시작 → 6 개월 단위 성능 모니터링 + 재학습 + 기준 미달 시 사업 종료 후 후속 사업 분리 |
| OEM 요구 변경으로 모델 재학습 | 외부 | 높음 | 중간 | 6 | OEM 변경 시 재학습 트리거 자동 + 1 개월 내 재학습 완료 SLA |

**AI 의사결정 책임 매트릭스 추가**:

AI 시스템이 출력하는 결정에 대해 **누가 최종 결정권 (Final Decision)·거부권 (Veto)·감사권 (Audit)·정보 수신 (Information Only)** 을 가지는지 명시.

| AI 의사결정 | 작업자 | QA·검사원 | 생산팀장 | OEM 감사관 | AI 운영자 |

**Final Decision Maker** — AI 출력을 수용·기각·수정할 최종 권한 + 결과 책임. **Veto Right** — AI 출력을 즉시 차단할 권한 + 차단 사유 기록 의무. **Audit Right** — 사후 감사를 위한 로그 열람권. **Information Only** — 결정 사실 통보만.

| RAG 검색 정확도 (Recall@5) | 정답 사례가 검색 결과 Top-5 에 포함된 비율 | (Top-5 정답 포함 건수 / 평가 표본 건수) × 100 | 분기 별 표본 100 건 수기 평가 | ≥ 85% | 분기 1 회 |
| 후보 신뢰도 (3 단계) | 검색 유사도 기반 후보별 신뢰도 점수 | 코사인 유사도 (벡터 임베딩 기반) × 100 | 검색 결과 즉시 산출 | 후보 1 ≥ 85, 후보 2 ≥ 80, 후보 3 ≥ 75 | 실시간 |

| 설계 재작업률 | 확정 후 재작업 발생 건수 비율 | (재작업 건수 / 전체 확정 건수) × 100 | 변경 이력 분석 | ≤ 7% | 월 1 회 |
| 품질 일관성 (설계 편차) | 동일 사양에 대한 설계 파라미터 표준편차 | σ (압하율·온도 등) | 분기 통계 | ±5% 이내 | 분기 1 회 |
| Mill Sheet OCR 인식 정확도 | 정확 추출 항목 수 / 전체 추출 대상 | (정확 항목 / 17 필수 항목 × 건수) × 100 | 원본 대조 검증 | ≥ 95% | 매주 (초기), 월 1 회 (안정화) |

#### 그림 31 신뢰도 (95/92/88%) 의 정의 명시

그림 31 의 "후보 1: 신뢰도 95% / 후보 2: 92% / 후보 3: 88%" 가 **검색 유사도 (cosine similarity) 기반 점수** 임을 명시. 사용자가 어느 후보를 채택할지 의사결정 기준을 본문에 추가:

- 신뢰도 85~89%: 베테랑 검수 후 채택
- 신뢰도 < 85%: 추천 거부 또는 전면 재검토

**RACI 정의**:
- **R (Responsible)** — 실행 책임자 (복수 가능)
- **A (Accountable)** — 결과에 대한 최종 설명 책임자 (1 명 한정)
- **C (Consulted)** — 사전 협의 대상
- **I (Informed)** — 결과 통보 대상
